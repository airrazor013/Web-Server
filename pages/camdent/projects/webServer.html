Web Server

In order to host this website, I developed my own web server using Node.js and Express framework.

The server was designed to be able to provide hosting to any number of different web domains. To provide each unique website hosted on the server the proper pages, I took advantage of Express' application-level middleware.

When an inbound request is received on the server, the domain that the request was sent from is analyzed to determine what directory the appropriate html files are located in. Once the directory was chosen, the url path specifies what html file to display. If the html file does not exist, then a 404 page is automatically displayed.

I chose to set the server up like this because it offered an easy way to add new pages to the website. Simply add a web page to the appropriate folder and the server will handle the rest. Using this configuration, there is no need to create a unique endpoint for each web page.
